## 2026-07-16 自动备份异常退出恢复后端

简要描述：

自动备份现在能识别异常退出、按设置调度完整归档，并为恢复操作保留当前工作区的保护副本。

详细描述：

- 每次桌面端启动都会写入独立的运行状态；上次仍处于运行中且存在可读自动备份时，启动端可提示用户选择恢复或正常打开。只有实际退出命令会标记为正常结束，关闭窗口隐藏到托盘不会误判为正常退出。
- 自动备份目录因权限、路径或读取异常无法检查时，启动不会被阻断：本次会话仍会标记为运行中，恢复提示默认关闭，并向界面返回可选的检查警告。
- Rust 后端按自动备份开关、15/30/60 分钟间隔、工作区数据库变更和保留份数决定是否创建归档；仅修改应用设置不会被误认为知识库内容变更。
- 恢复会先在同一自动备份目录生成带 `protection-` 前缀的完整保护归档，保护归档不会参与自动保留清理。保护归档失败会明确警告界面，但仍继续使用有效自动备份恢复；损坏、不可读或在导入中失败的自动备份都不会替换当前内容。
- 新增 `begin_auto_backup_session`、`run_auto_backup` 与 `restore_latest_auto_backup` 桌面命令。无论通过前端完成保存后退出，还是托盘无法通知前端而直接退出，都会在退出前持久化正常结束状态。
- 前端存储边界已为三条命令补齐 CamelCase 回执类型与调用；浏览器模式明确不执行归档或恢复，恢复检测安全返回“无需提示”。
- 工作区 store 已提供自动备份设置保存 action：只更新独立应用设置，保留关闭方式、主题和 MCP 配置，不写入工作区快照、不影响撤销历史或待保存内容。

验证情况：

## 2026-07-15 1.2.3 Windows 文件拖放栈溢出修复

简要描述：

修复 Windows 桌面端拖入文件时可能立即退出的问题。

详细描述：

- 根因位于本地资源写入：文件复制函数在 Rust 主线程栈上创建了 1 MiB 缓冲区，叠加既有调用栈后触发 `0xc00000fd` 栈溢出。
- 缓冲区现改为堆分配；DOCX、PDF 和普通附件继续复用同一套受控的资源写入、哈希与去重逻辑，不改变左侧顶级页面、正文子页面和附件落点规则。
- 移除了排查期间加入的临时日志，不会记录用户拖入的文件路径或内容。

验证情况：

- 已按 red → green 验证资源流缓冲区不再占用主线程栈；完整 Rust 测试 185 项通过。
- 相关前端测试 71 项通过，`npm run build` 通过，`npm run lint` 为 0 error、5 条既有 Hook warning。
- 调试版已由用户实际拖入 Word 复测通过，未再闪退。
