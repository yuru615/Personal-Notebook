## 1. 设计结论

思维导图采用“结构化导图编辑器”方案：

- 文档页中插入 `mindmap` 块。
- 块内只保存 `mindmapId`。
- 导图正文数据独立保存在 `MindmapRecord`。
- 点击块卡片进入独立思维导图编辑页。
- 导图核心用 React 结构化编辑器实现，不复用白板 legacy iframe 或白板自由画布对象模型。
- 复用白板周边基础设施：入口卡片、独立页面壳、标题栏、返回页面、资产存储、预览、备份导入导出模式。

这份设计对 2026-06-18 基线做一次收紧：此前“共享白板内核”的表述容易被理解成复用白板 legacy 画布核心。实际实现边界应为“共享产品基础设施，不共享自由白板业务核心”。

## 2. 用户确认的范围

本轮已确认：

- 选择结构化导图方案，不走白板对象拼导图。
- MVP 优先自动排版，不做自由拖拽摆放。
- 布局模式不是单选，三个都要做成可选项。
- 第一版重点是把导图编辑体验做稳，而不是追求复杂主题或自由布局。

## 3. MVP 功能

### 3.1 导图入口块

文档命令菜单中提供“思维导图”块。

插入后：

- 新建一个 `MindmapRecord`。
- 在当前文档插入一个 `mindmap` 卡片块。
- 卡片显示导图标题、更新时间和缩略图。
- 点击卡片进入导图编辑页。

### 3.2 独立导图编辑页

导图编辑页复用现有结构化画布页面壳：

- 顶部显示返回页面按钮。
- 标题可编辑。
- 显示来源页面。
- 主区域显示导图画布。
- 后续导出、布局设置、视图适配等操作放入导图页工具栏。

### 3.3 节点编辑

MVP 支持：

- 编辑节点文字。
- 选中节点。
- `Enter` 新增同级节点。
- `Tab` 新增子级节点。
- `Delete` 或空文本退格删除当前节点。
- 根节点不可删除。
- 删除节点时同时删除其全部子节点。
- 支持折叠/展开子节点。

暂不做：

- 节点备注。
- 节点图标。
- 节点标签。
- 多选批量操作。
- 节点自由拖动摆放。

### 3.4 布局模式

每个导图保存自己的布局模式。

新增字段建议：

```ts
type MindmapLayoutMode = 'balanced' | 'right' | 'outline'

interface MindmapRecord {
  layoutMode?: MindmapLayoutMode
}
```

三种模式：

- `balanced`：中心左右展开，作为默认模式。适合标准思维导图。
- `right`：只向右展开。适合流程、层级、任务拆解。
- `outline`：大纲式纵向展开。适合知识笔记和目录结构。

布局模式切换后：

- 不改变节点树数据。
- 不改变节点文本。
- 只改变布局计算和渲染方式。
- 已折叠节点继续保持折叠状态。

### 3.5 自动布局

布局计算应保持纯函数，输入 `MindmapRecord`，输出渲染节点和连接线。

要求：

- 支持任意深度父子节点。
- 同级节点按 `order` 排序。
- 隐藏折叠节点的子树。
- 输出节点坐标、连接线端点、画布边界。
- 三种布局共用节点树遍历逻辑，分别处理坐标排列。

第一版不做复杂避让算法。只要保证常见节点数量下不重叠、可读、可滚动缩放即可。

## 4. 白板复用边界

### 4.1 复用

可以复用或沿用：

- `CanvasEntryCard` 入口卡片模式。
- `StructuredCanvasPage` 独立编辑页外壳。
- 独立资产集合存储方式。
- `WorkspaceSnapshot.mindmaps` 备份字段。
- 右上角页面操作菜单的组织方式。
- 预览图生成思路。
- 未来可抽取的视口能力：缩放、平移、适配视图。

### 4.2 不复用

不复用：

- 白板 legacy iframe。
- 白板自由对象数组作为导图主数据。
- 白板任意连线系统。
- 白板图形绘制工具。
- 白板对象拖拽、吸附、层级、组合等自由画布规则。

原因：

- 思维导图的真实数据是节点树，不是画布对象。
- 导图连线由父子关系生成，不应该手动画任意线。
- 复用自由画布核心会让自动布局、折叠展开和快捷键变复杂。

## 5. 数据模型

现有 `MindmapRecord` 与 `MindmapNode` 基本可继续使用。

建议补充：

```ts
export type MindmapLayoutMode = 'balanced' | 'right' | 'outline'

export interface MindmapRecord {
  id: MindmapId
  title: string
  rootNodeId: string
  nodes: Record<string, MindmapNode>
  layoutMode: MindmapLayoutMode
  viewport: {
    x: number
    y: number
    zoom: number
  }
  createdAt: string
  updatedAt: string
}
```

`MindmapNode` 保持：

```ts
export interface MindmapNode {
  id: string
  parentId: string | null
  text: string
  order: number
  side?: 'left' | 'right'
  collapsed?: boolean
}
```

说明：

- `side` 主要服务 `balanced` 布局。
- `collapsed` 服务折叠展开。
- `layoutMode` 放在导图级别，而不是页面级别。

## 6. 交互细节

### 6.1 快捷键

- `Enter`：当前节点后新增同级节点。
- `Tab`：给当前节点新增子节点。
- `Shift + Tab`：可作为后续“提升层级”，MVP 可暂缓。
- `Delete`：删除选中节点。
- `Backspace`：节点文本为空时删除节点。
- `Esc`：退出文本编辑，保留节点选中。

### 6.2 工具栏

导图页工具栏第一版包含：

- 布局模式切换：左右导图、右侧导图、大纲导图。
- 适配视图。
- 缩放比例显示。

暂不放：

- 主题市场。
- 复杂样式面板。
- 导出图片。
- 演示模式。

### 6.3 节点菜单

节点上可提供轻量操作：

- 添加子级。
- 添加同级。
- 折叠/展开。
- 删除。

菜单点击空白区域应关闭，保持和现有编辑器浮层规则一致。

## 7. 测试策略

需要覆盖：

- `mindmapModel`：新增同级、子级、删除子树、折叠展开、布局模式切换。
- `mindmapLayout`：三种布局模式、深层节点、折叠节点不参与布局。
- `MindmapCanvas`：快捷键触发、节点编辑、菜单按钮、布局模式切换。
- `App`/store：插入 `mindmap` 块、保存、重新打开、备份导入导出包含 `mindmaps`。
- 预览：不同布局模式下能生成稳定缩略图。

## 8. 非目标

MVP 不做：

- 白板式自由拖拽摆放。
- 任意节点之间手动画连接线。
- 复杂主题样式。
- 图片节点、附件节点。
- 与页面块互相引用。
- 导出 PNG/SVG。
- 大纲文本双向导入导出。
- 协作、评论、提醒。

这些功能后续可以单独进入二期。

## 9. 实施顺序建议

1. 修正命令菜单中文文案。
2. 补齐 `MindmapRecord.layoutMode` 和兼容旧数据的默认值。
3. 扩展 `mindmapModel`：折叠展开、布局模式切换、快捷键需要的节点操作。
4. 重写 `mindmapLayout`：支持 `balanced`、`right`、`outline`。
5. 优化 `MindmapCanvas`：选中态、文本编辑、快捷键、节点菜单。
6. 给导图页加工具栏：布局切换、适配视图、缩放显示。
7. 更新预览图生成。
8. 补测试并做浏览器验证。

## 10. 验收标准

第一版完成时，应满足：

- 用户能从命令菜单插入思维导图块。
- 点击块能进入导图编辑页。
- 能连续创建同级和子级节点。
- 能删除非根节点。
- 能折叠和展开分支。
- 能在三种布局之间切换，切换后节点数据不丢失。
- 关闭页面后重新打开，导图内容、布局模式和折叠状态仍然存在。
- 导图卡片能显示可识别缩略图。
- 所有新增 UI 文案为中文。
