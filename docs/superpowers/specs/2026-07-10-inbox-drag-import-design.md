# 收件箱拖拽导入设计

## 目标

让用户把操作系统中的文件直接拖入知栖窗口，文件自动保存到本地资产库，并追加到收件箱。

## 第一期开关边界

- 接收操作系统文件拖拽，不接管页面树、数据表和块排序等站内拖拽。
- 图片、音频、视频复用现有媒体块；其他文件新增轻量附件块。
- 每次拖入作为一批内容，收件箱中以“拖拽导入 · 时间”作为批次标题。
- 写入前复用现有收件箱创建、资产写入和工作区持久化链路。

## 不在本期

- Markdown 转换为可编辑页面。
- PDF 预览、PDF 全文搜索。
- 在资源管理器中显示原文件、调用系统默认程序打开原文件。
- 拖入目标页面或指定块位置。

## 交互

当文件被拖入应用窗口时，显示半透明全窗口提示“松开以导入到收件箱”。松开后提示消失；导入成功后数据出现在收件箱，不强制跳转页面。非文件拖拽完全保持当前行为。

## 数据流

`AppShell` 识别外部文件拖拽并把 `File[]` 交给应用层。应用层将文件写入资产库并转换为块：媒体文件映射到现有媒体块，其他文件映射到附件块。store 通过统一的收件箱追加操作写入标题、块和尾部空段落，再由现有 repository 持久化。

## 验收

- 拖入图片、音频或视频后，收件箱出现对应可显示的媒体块。
- 拖入 PDF、文档或压缩包后，收件箱出现带文件名和类型的附件块。
- 多个文件保持拖入顺序。
- 站内页面排序、块排序和数据表拖拽不触发导入遮罩或收件箱写入。
- 资产可随工作区备份、页面包和孤立资源清理正确处理。
