# 高中语文教师个人备课与知识沉淀工作台设计

## 1. 目标

制作一份可直接导入知栖、可重复复制的 `.zhiqi` 页面包模板，服务高中语文教师的日常备课、教学执行、资源积累、匿名学情观察和教学复盘。

模板采用高一上册第七单元“自然情怀”作为完整示例，覆盖《故都的秋》《荷塘月色》《我与地坛（节选）》《赤壁赋》《登泰山记》。模板展示知栖现有页面、块、数据表、白板、思维导图、同步块、媒体和搜索能力，但不扩展为教务或学生成绩管理系统。

## 2. 交付边界

本次交付包括：

- 一棵以“高中语文教师工作台｜高一上学期”为根的完整页面树。
- 三张数据表、两块白板、两张思维导图和两组同步块。
- 原创或自行制作的轻量媒体示例，不打包受版权保护的教材扫描件、商业课件或朗读音频。
- 可维护的模板数据构建器和可重复执行的生成脚本。
- 可直接导入的 `public/templates/high-school-chinese-teacher-workbench.zhiqi` 页面包。
- 模板结构、引用完整性、内容边界、真实导入和重复导入验证。

本次不包括：

- 模板中心、模板列表、预览页或“一键使用模板”界面。
- 对现有“欢迎使用知栖”默认工作区的修改。
- 新增第三方依赖。
- 真实学生姓名、成绩、排名或个人辅导档案。
- 页面包不能携带的自定义页面属性定义；状态、日期、班级和课型等结构化信息统一放在数据表中。

## 3. 使用方式

用户从知栖“导入页面包”入口选择模板文件。导入完成后，整棵页面树作为新的顶层页面加入当前工作区，不覆盖已有内容。现有导入逻辑会为页面、数据表、白板、思维导图、同步块和媒体重新分配 ID，并重写包内引用，因此同一模板可以重复导入为相互独立的副本。

导入后的首次使用流程为：

1. 打开“00 模板使用说明”。
2. 修改学年、学期、任教班级和示例日期。
3. 在教学任务库中调整课程与截止日期。
4. 把“待补充”的资源记录替换为自己的资料。
5. 删除不适用的课文、班级和示例提示后开始使用。

## 4. 页面树

## 5. 日常使用动线

1. 教师每天从“01 教师工作台”查看本周课程、未完成任务、近期教学和待整理资料。
2. 备课时从“第七单元｜自然情怀”进入对应课文页，查看单元目标、知识卡片、教学流程和分层作业。
3. 上课前在教学任务库更新状态，上课时打开《荷塘月色》课堂流程白板或相关思维导图。
4. 课后将共性问题写入学情观察库，将有效资料和活动沉淀进教学资源库。
5. 完成课后复盘；单元结束后在单元复盘中提炼可复用经验。

## 6. 页面内容设计

### 6.1 模板使用说明

页面使用待办块提供初始化清单，使用带背景色的段落区分“必须修改”“可以删除”和“保留作参考”的内容，并说明页面包再次导出和完整工作区备份的区别。

### 6.2 教师工作台

首页预置：

- 示例学期：2026—2027 学年第一学期。
- 匿名班级：高一（3）班、高一（6）班。
- 本周教学重点、临近课程和四类待办：备课、授课、批改、复盘。
- 教学任务库的内联入口。
- 第七单元核心目标同步块。
- 第七单元、五篇课文、资源库、学情观察库和教研灵感收件箱的页面入口。

### 6.3 教学规划

“学期教学地图”嵌入学期思维导图，展示阅读与鉴赏、表达与交流、梳理与探究、整本书阅读四条主线。

“单元总览”预置：

- 人文主题与语文核心素养目标。
- 核心问题：“作者如何借自然景物表达生命感受？”
- 五篇课文的联系与差异。
- 三周任务链：朗读感知、文本细读、比较阅读、微写作、单元成果。
- 形成性评价与终结性评价。
- 单元教学设计白板、知识导图和评价量规普通表格。

### 6.4 五篇课文备课页

五篇课文统一包含：教材定位、匿名学情分析、学习目标、教学重点与难点、课前准备、教学流程、核心问题链、板书设计、分层作业和课后复盘入口。

《荷塘月色》作为最完整示范课，预置两课时流程、朗读设计、景物层次、语言赏析、通感知识卡片、情感变化、课堂流程白板和微写作任务。其余四篇保持完整结构，并分别突出：

- 《故都的秋》：景物选择、色彩语言和“清、静、悲凉”。
- 《我与地坛（节选）》：生命体验、母亲形象和文本细读。
- 《赤壁赋》：重点文言知识、主客问答和情感变化。
- 《登泰山记》：游踪、时空层次和景物描写。

课文页综合使用标题、段落、带背景色段落、项目列表、编号列表、待办、普通表格、代码块式投屏文本、富文本颜色、页面提及、子页面块、同步块和页面目录。

### 6.5 资源与知识沉淀

“语文知识卡片”下的六张知识页提供定义、判断方法、课内例句说明、易混点和迁移练习。课文页通过页面提及指向知识卡片，形成双向链接。

“写作素材库”提供自然景物观察、感官描写、情感线索和微写作提示，使用普通页面和教学资源库记录配合，而不建立第四张数据表。

### 6.6 作业、学情和复盘

“作业设计与评价”包含基础巩固、文本细读、比较阅读、微写作、分层作业和单元评价量规。

“作业共性问题台账”和“班级教学观察”引用同一张学情观察库，通过视图和筛选区分用途。模板只记录班级级别的匿名观察。

“课后复盘”提供目标达成、课堂证据、意外情况和下次调整四部分固定结构；“单元复盘”负责有效设计、共性困难、资源复用和下轮教学建议；“教研灵感收件箱”保存尚未归类的案例、问题和活动构思。

## 7. 数据表设计

### 7.1 教学任务库

字段：任务名称、状态、任务类型、所属课文、班级、开始日期、截止日期、优先级、备注。

视图：

- “全部任务”：表格视图。
- “备课看板”：按状态分组的看板视图。
- “教学日历”：按上课或截止日期显示的日历视图。
- “单元进度”：使用开始与截止日期的甘特图视图。
- “本周待办”：筛选未完成且位于示例周范围的表格视图。

预置 22 条记录，覆盖备课、资料准备、授课、作业、资源整理和复盘。状态选项为未开始、进行中、待反馈、已完成；优先级为高、中、低。

### 7.2 教学资源库

字段：资源名称、资源类型、适用课文、使用场景、来源、标签、准备状态、备注。

预置 16 条记录，覆盖教案、课件、朗读、图片、视频链接、论文评论、练习题和写作素材。未实际随包附带的版权资源统一标为“待补充”，不得伪装成已包含附件。

### 7.3 学情观察库

字段：问题或观察、班级、所属课文、类型、出现频率、课堂证据、改进策略、跟进状态、复查日期。

视图：全部观察表格、按跟进状态看板、复查日历。预置 8 条匿名班级观察，覆盖修辞辨析、朗读证据、文言结构和写景表达等典型问题。

## 8. 白板设计

### 8.1 第七单元教学设计白板

白板按从左到右的链路组织：单元主题 → 核心问题 → 文本研读 → 学习活动 → 学习成果 → 评价与复盘。五篇课文作为可移动便签，现代散文与古代山水文章使用不同色组；核心节点使用箭头连接，并保留“课堂生成”空白便签区。

### 8.2 《荷塘月色》课堂流程白板

白板划分第一课时、第二课时、板书布局和课堂生成四个区域。第一课时包含情境导入、整体朗读、结构梳理和景物层次；第二课时包含语言赏析、通感探究、情景关系和微写作迁移。每个环节标注建议用时，并通过连线表示先后和反馈关系。

## 9. 思维导图设计

### 9.1 第七单元知识导图

中心节点为“自然情怀”，一级分支为现代散文、古代山水、阅读方法、表达知识和单元成果；二级节点覆盖五篇课文、抓景物特征、理写景层次、品味语言、情景交融、通感、移步换景、虚实结合、比较阅读、赏析札记和写景散文。

### 9.2 《荷塘月色》文本细读导图

中心节点为“荷塘月色”，一级分支为行文结构、景物层次、语言特点、情感变化和主题理解；二级节点覆盖月下荷塘、荷塘月色、比喻、拟人、通感、叠词、“颇不宁静”到暂得宁静，以及背景材料的解释边界。

## 10. 同步块、关系与搜索

两组同步块为：

1. “第七单元核心目标”：出现在教师工作台、单元总览和五篇课文页。
2. “课后复盘四问”：出现在五篇课文页和课后复盘页。

课文页提及相关知识卡片，作业和学情内容链接回对应课文，单元复盘链接到任务、资源和学情入口。页面、白板、导图和数据表使用一致的课文名、核心概念和教学关键词，确保全局搜索可以演示跨内容类型检索。

所有页面关系必须留在根页面树内。模板不得引用包外页面，也不得创建引用了包外实例的同步块。

## 11. 媒体与视觉

根页面和单元页面使用知栖内置封面样式。页面包附带：

- 一张自行制作的《荷塘月色》意象示意图，作为图片块使用并填写替代文本和说明。
- 一份纯文本“朗读停连标记示例”，作为文件块使用。

教学资源库可以保存国家中小学智慧教育平台和人民教育出版社等公开站点链接，但链接仅作为入口，不抓取或打包站内版权内容。

## 12. 实现结构

模板构建器位于：

```text
src/domain/templates/highSchoolChineseTeacher.ts
```

它返回页面包所需的根页面 ID、页面、白板、数据表、思维导图、同步块和媒体清单。模板使用现有领域类型、数据表工厂、白板模型和思维导图模型，不在组件中复制领域逻辑。

生成脚本位于：

```text
scripts/generate-teacher-template.mjs
```

脚本通过 Vite 加载 TypeScript 模板构建器，使用项目已有 `jszip` 写出知栖 exchange archive v2 的 `manifest.json`、`payload.json`、`assets/manifest.json` 和媒体文件。脚本不增加第三方依赖。

`package.json` 增加：

```json
"template:teacher": "node scripts/generate-teacher-template.mjs"
```

最终产物固定为：

```text
public/templates/high-school-chinese-teacher-workbench.zhiqi
```

生成脚本在写文件前校验重复 ID、页面树连通性、块资源引用、数据表视图字段引用和媒体清单。任一校验失败时不发布目标文件；成功生成时先写临时文件，再替换目标文件，避免留下半成品。

## 13. 导入与错误处理

页面包继续使用现有存储层导入能力。导入前由现有 manifest 校验确认页面树、资源和同步块关系完整；导入过程为事务性写入，失败时不保留部分页面和资源。

模板生成和导入均不访问网络。外部链接失效不会影响页面包导入，媒体文件缺失则由生成校验或现有导入校验直接拒绝。

## 15. 成功标准

满足以下条件即视为完成：

1. 用户可以通过现有“导入页面包”入口导入模板，且不覆盖当前工作区。
2. 模板形成完整、连通、可编辑的教师工作台页面树。
3. 页面、数据表、白板、思维导图、同步块和媒体在导入后均可正常打开和编辑。
4. 同一模板可重复导入，副本间 ID 和引用互不冲突。
5. 模板内容具有真实高中语文教学语境，不包含学生隐私和未授权版权资料。
6. 自动测试、构建检查和 Windows 桌面端实际导入验证均通过。
