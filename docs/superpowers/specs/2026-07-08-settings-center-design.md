# 设置中心设计

- 日期：2026-07-08
- 状态：待评审
- 适用项目：知栖桌面端
- 主题：全量完整设置中心

## 1. 背景

知栖已经逐步形成了较完整的本地优先知识库产品形态：页面树、页面属性、搜索、白板、数据表、导入导出、桌面托盘、收件箱、同步块与关系化能力都已经开始成型。

但当前“设置”仍然是分散的：

- 一部分是工作区设置，例如收件箱页面、侧边栏布局、侧边栏宽度、星标置顶。
- 一部分是页面级显式值，例如全宽、小字号、字体、右侧目录。
- 一部分是桌面行为，例如托盘、关闭行为、后续要做的剪贴板捕获与快捷入口。
- 还有一部分属于维护动作，例如备份、恢复、资源清理、索引修复，目前还没有统一入口。

如果继续沿用“哪里有功能就把开关塞到哪里”的方式，后面一定会出现三个问题：

1. 用户不知道某个行为该去哪里改。
2. 相似设置分散在不同入口，后续维护成本会越来越高。
3. 新增桌面端、本地工作流、备份恢复能力后，应用缺少一个稳定的产品级配置中心。

因此需要建立一套完整的设置中心，把系统级偏好、工作区级偏好、页面默认规则和数据维护入口统一收拢。

## 2. 目标

本次设置中心设计的目标是：

1. 建立一个独立、完整、可持续扩展的设置中心。
2. 把已有的分散设置统一收口，而不是另起一套和现有能力脱节的新系统。
3. 明确系统级、工作区级、页面默认级三层设置边界。
4. 让后续方向 2 中的桌面能力也能自然纳入设置中心。
5. 保持本地优先和低风险迁移，不破坏已有工作区和页面数据。

## 3. 非目标

本次设计明确不做以下事情：

- 不把设置中心做成普通页面内容的一部分。
- 不把设置项全部变成“页面内可编辑块”，避免设置系统和知识内容混淆。
- 不在第一版中同时落地所有未来能力，例如完整快捷键系统、成熟通知中心、多窗口协同。
- 不强行改写已有页面的显式展示值。
- 不为“操作”额外发明一套 settings 数据结构，例如“立即备份”“立即恢复”“立即重建索引”不应持久化为设置值。

## 4. 核心产品决策

以下决策已经固定：

### 4.1 设置中心是应用级页面，不进入页面树

设置中心应作为应用内独立页面存在：

- 从顶部功能区的“更多”进入。
- 打开后仍复用现有应用外壳。
- 不进入“我的页面”。
- 不参与搜索。
- 不参与页面包导出。
- 不作为知识内容的一部分出现在页面树中。

这能避免“知识内容”和“应用配置”边界混乱。

### 4.2 设置中心采用“左侧分类 + 右侧内容”

不采用长单页，不采用多弹窗，不采用“基础设置 / 高级设置”拆分。

固定采用：

- 左侧分类导航
- 右侧当前分类内容

原因：

- 结构稳定，适合持续扩展。
- 比长单页更容易查找。
- 比“基础 / 高级”更符合当前要做“完整设置中心”的目标。

### 4.3 完整设置一起纳入

本次设置中心不是只收视觉偏好，而是完整纳入以下四类：

- 系统级
- 工作区级
- 页面默认级
- 数据与维护入口

### 4.4 页面默认值归属于工作区

“页面默认值”不作为独立第四套持久化结构，而是作为 `WorkspaceSettings` 的一部分存在。

原因：

- 它描述的是“这个工作区里以后新建页面默认长什么样”。
- 它不是系统级行为。
- 它也不是单页内容。

## 5. 信息架构

设置中心左侧固定为 8 个分类：

1. 通用
2. 外观与侧边栏
3. 编辑与页面默认
4. 搜索与知识组织
5. 导入导出
6. 桌面端
7. 数据与维护
8. 实验功能

右侧内容统一采用以下结构：

1. 顶部标题区
2. 设置分组区
3. 底部状态区
4. 危险操作区（仅在需要时显示）

其中：

- 顶部标题区展示分类名、简短说明、当前工作区信息。
- 设置分组区承载实际可改配置。
- 底部状态区展示“已自动保存 / 保存失败 / 需重启后生效”等状态。
- 危险操作区与普通开关分离，避免误操作。

## 6. 设置分类与条目范围

### 6.1 通用

用于承载最基础的应用与工作区通用信息：

- 应用名称
- 版本号
- 当前工作区路径
- 启动时打开上次工作区
- 启动后恢复上次页面
- 时间显示格式
- 语言预留
- 默认首页 / 启动页策略

### 6.2 外观与侧边栏

用于承载导航和整体视觉密度：

- 侧边栏布局：紧凑 / 经典
- 记住侧边栏宽度
- 默认侧边栏宽度
- 星标置顶区默认展开 / 折叠
- 我的页面区默认展开 / 折叠
- 共享页面区显示策略
- 左侧树是否单行省略
- 左侧更多按钮显示方式
- 全局滚动条粗细 / 颜色风格
- 紧凑模式总开关

### 6.3 编辑与页面默认

用于承载新页面默认规则和基础编辑偏好：

- 新页面默认全宽
- 新页面默认小字号
- 新页面默认字体
- 新页面默认显示右侧目录
- 新页面默认显示属性区
- 正文默认行高
- 标题块与特殊块默认间距策略
- 空白行提示词显示策略
- 回车创建新行时的默认块类型
- 页面属性默认预置项

### 6.4 搜索与知识组织

用于承载搜索结果形态和知识组织策略：

- 搜索结果分组显示
- 同页多命中是否全部展开
- 结果上下文长度
- 是否索引图片说明文字
- 是否索引音频 / 视频说明文字
- 是否索引白板文字
- 是否索引导图节点
- 是否显示命中来源标签
- 搜索回车跳转策略
- 收件箱页与新建笔记默认落点策略

### 6.5 导入导出

用于承载默认导入导出规则：

- 默认导出目录
- 导出全部时是否包含资源文件
- 导出页面包时是否包含子页面
- 导入冲突处理策略
- 导入后是否自动跳转
- 是否保留原文件路径信息
- Markdown / 页面包 / JSON 的默认导出偏好

### 6.6 桌面端

用于承载桌面应用行为：

- 关闭主窗口时最小化到托盘 / 直接退出
- 是否启用系统托盘
- 托盘快捷入口显示项
- 开机启动
- 全局快捷键速记开关
- 剪贴板捕获模式：关闭 / 手动 / 半自动
- 半自动捕获默认进入收件箱
- 新建笔记默认位置
- 通知开关

### 6.7 数据与维护

用于承载备份、恢复、诊断和修复：

- 自动备份开关
- 自动备份频率
- 自动备份保留数量
- 崩溃恢复开关
- 资源文件自检
- 孤立资源清理
- 数据库修复 / 重建搜索索引
- 导出完整工作区备份
- 从备份恢复
- 查看当前数据库与资源目录

### 6.8 实验功能

用于承载尚未完全收口但需要给用户选择权的能力：

- 引用块 / 同步块增强交互开关
- 新搜索体验开关
- 新关系化视图开关
- 剪贴板增强捕获实验项
- 多窗口 / 多工作区实验项

## 7. 持久化分层

设置必须拆成三层：

### 7.1 AppSettings

描述“这台电脑上的知栖怎么工作”。

典型内容：

- 关闭窗口行为
- 托盘开关
- 开机启动
- 通知
- 全局快捷键
- 剪贴板捕获模式
- 默认导出目录
- 全局滚动条风格

### 7.2 WorkspaceSettings

描述“这个工作区怎么工作”。

典型内容：

- 收件箱页面 ID
- 侧边栏布局与宽度
- 分组展开状态
- 搜索分组偏好
- 导入导出默认策略
- 自动备份频率
- 资源自检偏好
- 新建笔记默认落点
- 页面默认规则

### 7.3 页面显式值

描述“某一页当前已经被明确改成什么样”。

例如当前已有的：

- `isFullWidth`
- `isSmallText`
- `fontFamily`
- `showOutline`

这些不是“默认值”，而是页面自己的显式状态。

## 8. 优先级规则

渲染与行为优先级固定如下：

1. 页面当前显式值优先级最高
2. 页面没有显式值时，才读取页面默认值
3. AppSettings 不参与页面展示优先级，只影响应用行为

这意味着：

- 修改默认值不会批量改坏已有页面
- 默认值只影响之后新建的页面
- 老页面的显式配置继续保留

## 9. 什么要存，什么不存

### 9.1 需要持久化的

以下内容需要保存到设置结构中：

- 偏好
- 默认值
- 行为策略
- 分组展开状态
- 路径偏好
- 搜索显示偏好

### 9.2 不需要持久化的

以下内容属于动作，不属于设置：

- 立即导出
- 立即备份
- 恢复备份
- 重建索引
- 修复数据库
- 清理孤立资源

这些可以放在设置中心里，但应该以“操作按钮”存在，而不是写进 settings 数据。

## 10. 导出、备份与迁移边界

### 10.1 AppSettings

- 不进入页面包
- 不进入工作区导出
- 不跟随知识库迁移到另一台电脑

### 10.2 WorkspaceSettings

- 进入完整工作区备份
- 跟随知识库迁移

### 10.3 Page Defaults

- 作为 `WorkspaceSettings` 的一部分进入完整工作区备份

### 10.4 单页面导出包

- 不带整套设置中心配置
- 只带页面内容所需的最小信息

这样可确保：

- 工作区自己的习惯会一起迁移
- 电脑自己的行为设置不会被强行带到新设备

## 11. 交互规则

### 11.1 普通设置

普通偏好项使用即时保存：

- 用户修改后界面立即更新
- 后台自动保存
- 页面底部显示保存状态

### 11.2 路径类设置

路径类设置采用：

- 只读路径显示
- “选择”按钮调起系统选择器

### 11.3 危险操作

危险动作必须：

- 二次确认
- 显示影响范围
- 在执行中提供明确状态
- 完成后给出结果摘要

### 11.4 需重启项

对少数系统级设置，应明确标记：

- 需重启应用后生效

## 12. 错误处理

错误处理分为三类：

### 12.1 普通偏好保存失败

例如侧边栏布局、默认字体、搜索分组。

处理原则：

- 页面先响应用户操作
- 保存失败时显示“保存失败”
- 保留重试入口
- 不锁死整页

### 12.2 系统级设置失败

例如托盘、开机启动、全局快捷键、通知权限。

处理原则：

- 明确说明失败原因
- 区分“系统不支持”和“保存失败”
- 不影响知识库内容本身

### 12.3 危险操作失败

例如恢复备份、重建索引、修复数据库、清理资源。

处理原则：

- 操作前确认
- 操作中展示状态
- 失败时说明失败阶段
- 能回滚则回滚，不能回滚则提前提示风险

## 13. 危险操作统一规则

所有高风险动作统一遵守：

1. 恢复前建议先导出当前完整备份
2. 删除类动作必须二次确认
3. 影响较大时显示受影响范围
4. 完成后给出结果摘要

## 14. 兼容与迁移策略

本次设置中心只能做“向前兼容扩展”，不能推倒重来。

启动时应：

- 继续读取现有 `WorkspaceSettings` 老字段
- 统一 normalize 到新结构
- 对缺失新字段补默认值
- 保留已有页面显式值语义

这意味着：

- 老工作区可以平滑升级
- 旧数据不会因为设置中心扩容而失真

## 15. 第一版实现范围

设置中心虽然设计为完整结构，但实现应分期。

### 15.1 第一期：骨架与现有能力收拢

第一期只做最稳、最贴近现有代码的部分：

- 设置中心独立页面
- 左侧分类导航
- `AppSettings / WorkspaceSettings / Page Defaults` 三层结构
- 现有设置正式收口：
  - 侧边栏布局
  - 侧边栏宽度
  - 收件箱页
  - 星标置顶相关偏好
  - 新页面默认全宽 / 小字号 / 字体 / 目录
  - 桌面端关闭行为、托盘开关等低风险项

### 15.2 第二期：工作区偏好补全

- 搜索分组与命中策略
- 导入导出默认策略
- 页面属性默认预置项
- 自动备份与恢复入口
- 数据维护页

### 15.3 第三期：桌面端本地工作流增强

- 全局快捷键速记
- 剪贴板捕获模式
- 通知
- 多窗口 / 实验功能
- 更强的本地文件联动

## 16. 第一版优先级

第一版优先完成以下五组：

1. 设置中心壳
2. 通用
3. 外观与侧边栏
4. 编辑与页面默认
5. 桌面端（仅收已有和低风险项）

原因：

- 这些能力已经有现成的产品基础
- 变更边界清晰
- 最容易先做出一个真实可用、后续又能继续扩展的设置中心

## 17. 实现边界建议

为了避免第一版“乱长”，实现时应遵守以下边界：

- 复用现有应用外壳，不另起一套设置窗口
- 能沿用现有 store 与 repository 就不新造状态系统
- 能沿用现有页面默认字段语义就不重做页面模型
- 不为设置中心单独发明“内容块化编辑器”
- 不把维护动作误做成持久化设置值

## 18. 验收标准

以下全部满足，可视为第一期设置中心成立：

1. 用户可从稳定入口进入设置中心。
2. 设置中心以“左侧分类 + 右侧内容”稳定工作。
3. 已有侧边栏、页面默认、桌面端低风险设置被收拢进统一入口。
4. 页面默认值只影响新页面，不回写旧页面显式值。
5. AppSettings 与 WorkspaceSettings 边界清晰，不互相污染。
6. 危险操作与普通设置视觉分离。
7. 老工作区升级后不会丢失现有设置与页面显示语义。

## 19. 结论

这次设置中心不应该只是“再补几个开关”，而应该成为知栖的应用级配置底座。

它的核心判断是：

- 设置中心是应用能力，不是知识内容。
- 系统级、工作区级、页面默认级必须硬分层。
- 完整设计一次定好，落地时按三期推进。
- 第一版先做结构和已有能力收口，不抢跑高风险桌面特性。

这样做最稳，也最适合知栖后续继续长成功能更完整的本地优先桌面知识库。
