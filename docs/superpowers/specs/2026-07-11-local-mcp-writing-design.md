# 知栖本机 MCP 写入接入设计

## 目标与范围

第一期在知栖桌面应用内提供仅限本机的 MCP 服务，让已授权 AI 客户端把生成内容直接保存到当前工作区。覆盖支持 MCP 的桌面 AI 客户端，首次授权后允许直接新增写入。

支持：文本块、普通表格、图片/视频/音频/附件、数据表、白板和思维导图。

不支持：云端或局域网访问、账号与跨设备同步、网络 API、通用 Skills 运行时，以及删除、覆盖、移动页面、清理资产等高风险操作。

## 方案选择

采用“桌面应用内置本机 MCP 服务”。知栖启动并由用户在设置中启用后，只在 `127.0.0.1` 提供 MCP Streamable HTTP 端点；端口与高熵 Bearer 令牌均为本机配置。

不采用独立 MCP 进程直接读取 SQLite：这会绕过现有 `StorageState` 的事务与资产引用维护，并可能与应用争用数据库。也不以 Skills 作为数据通道：Skills 只能说明某一 AI 客户端如何使用工具，不能形成跨客户端写入协议。

## 架构与数据流

1. 用户在“设置 → 外部 AI 接入”启用 MCP；应用随机生成端口与令牌。
2. 用户复制 MCP 配置到 AI 客户端。
3. 服务仅接受回环地址和有效 Bearer 令牌。
4. 工具处理器调用应用内 `StorageState` 与内容导入服务，不直接写 SQL 或资产目录。
5. 导入服务原子创建资产、复杂对象和页面块；失败时不留下半成品引用。
6. 成功后服务向前端发出工作区更新事件，使已打开页面刷新。

禁用开关或重新生成令牌立即使旧配置失效。

## 授权与能力边界

首次启用时明确告知用户：已配置的本机 AI 客户端可搜索页面、读取用户允许读取的页面内容，并向指定页面新增内容。令牌只授予 `read_pages` 与 `append_content`；没有删除、替换、移动、重命名、导出、清理资产或任意文件访问权限。

所有创建类工具要求 `pageId` 或明确的父页面 ID。AI 可以使用搜索工具获取候选页，但不得依标题模糊匹配后静默写入。

## MCP 工具

- `search_pages({ query, limit? })`：返回页面 ID、标题、父页面 ID 与摘要。
- `get_page({ pageId })`：返回页面元数据和可读块摘要，不返回资产二进制。
- `create_page({ parentId, title, icon? })`：创建空子页面。
- `append_content({ pageId, content })`：追加受控内容块。

`content` 是受限联合体：`markdown`、`table`、`asset`、`dataTable`、`whiteboard`、`mindmap`。服务端校验尺寸、字段、目标页面与引用关系；未知或无法转换的内容拒绝写入，不做猜测。

## 资产与复杂内容

资产可由小型 Base64 数据或 MCP 客户端显式暴露的本机文件路径提供。服务总是复制数据到知栖资产库，再创建 image、video、audio 或 file 块，不保留原始路径引用。Base64 仅用于图片和小附件，单项上限固定为 20 MB；视频等大文件只支持本机文件导入。远程 URL 下载不进本期。

外部 AI 不得提交知栖内部 `snapshot`。白板、思维导图和数据表接受稳定的语义输入，再由导入服务转换为现有编辑器快照：

- 数据表：标题、列定义、记录值。
- 思维导图：根主题、递归子主题和简短注释。
- 白板：文本/便签/矩形节点及带标签连线；服务负责布局和坐标。

每种复杂内容都先创建独立领域记录，再追加对应页面块，确保独立页面入口和块入口均可继续编辑。

## 体验、错误与审计

设置页提供启用/禁用、连接状态、复制配置、重新生成令牌和最短使用说明。令牌默认遮蔽，主动复制时才展示。知栖未启动时，AI 客户端得到“请启动知栖或检查 MCP 设置”的错误。

无效令牌、非回环来源或禁用状态不泄露页面信息。找不到页面、内容不支持、校验失败或资产过大返回可操作错误。同一次 `append_content` 全成或全败。每次成功写入仅本机记录时间、客户端标识、工具名、目标页面 ID 和创建对象 ID。

## 验证与分期

Rust 单测覆盖令牌与回环校验、权限拒绝、内容规范化、事务回滚和复杂内容转换；前端测试覆盖设置、配置复制、令牌重置和写入通知；内存 SQLite 集成测试验证各类内容能导出并还原 JSON 备份。

第一子阶段交付服务生命周期、授权设置、页面搜索、创建页面及 Markdown/普通表格追加。第二子阶段加入资产导入和数据表。第三子阶段加入受限语义白板与思维导图转换。每阶段均保持既有导入导出和资产清理测试通过。
