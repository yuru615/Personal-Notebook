# 页面属性行内编辑设计

**日期：** 2026-07-05

**目标：**
把页面属性编辑从浏览器 `prompt` 改成更顺手的行内编辑体验，优先解决日期属性的可用性问题，同时统一文本、标签、状态、日期四类属性的编辑方式。

## 范围

- 仅调整普通页面头部中的 `PagePropertiesPanel`
- 不新增依赖
- 不改后端数据结构
- 不引入复杂弹窗系统

## 设计原则

- 保持轻量：优先复用现有面板区域，在属性行内完成编辑
- 保持统一：四类属性都走同一套“点击值 -> 进入编辑态 -> 保存/取消”的交互
- 保持最小改动：只改属性面板相关组件、样式和测试
- 保持兼容：沿用现有 `onSetValue` 和属性值结构

## 交互设计

### 1. 单行编辑态

- 同一时间只允许一个属性进入编辑态
- 点击属性值后，当前行切换为编辑控件
- 支持：
  - `Enter` 保存
  - `Escape` 取消
  - 失焦保存

### 2. 日期属性

- 使用原生 `<input type="date">`
- 进入编辑态后直接显示日期选择器
- 提供清空按钮
- 保存值仍为现有字符串格式，空值保存为 `null`

### 3. 状态属性

- 若存在 `definition.config.options`，展示轻量选项列表
- 点击选项立即保存
- 若没有配置选项，保留当前最小回退逻辑

### 4. 文本/备注属性

- 使用单行输入框
- 初始值回填当前内容
- 空字符串保存为 `null`

### 5. 标签属性

- 使用轻量文本输入
- 仍采用“逗号分隔”编辑方式
- 保存时裁剪空白并过滤空项
- 先不做复杂 tag 芯片编辑

## 视觉设计

- 编辑控件保持在属性行内，不额外弹出模态框
- 输入框、日期框、选项列表沿用当前页面属性的紧凑风格
- 日期清空按钮和保存中的状态保持低干扰

## 技术设计

### 组件

- 继续使用 `src/components/editor/PagePropertiesPanel.tsx`
- 在组件内部新增当前编辑属性状态
- 为不同属性类型渲染不同编辑控件

### 文案

- 在 `src/ui/copy.ts` 补充：
  - 日期清空
  - 日期占位/空值
  - 状态选择
  - 标签编辑提示（若现有文案不够）

### 测试

- 先补失败测试，再改实现
- 重点覆盖：
  - 日期属性进入日期编辑态
  - 日期选择后触发保存
  - 日期清空
  - 状态属性使用配置选项保存
  - 文本属性回车保存
  - 标签属性按逗号拆分保存

## 风险与处理

- 原生日期控件样式可塑性一般：
  - 接受这个限制，换取稳定和最小改动
- 失焦保存可能和点击选项交互互相影响：
  - 优先用最小事件处理方案，必要时按属性类型分别处理

## 推荐落地

采用“统一行内编辑 + 日期原生选择器”的最小实现。

这条路径能明显提升体验，又不会把页面属性编辑做成一个过重的新子系统。
