# 知栖桌面应用图标设计

- 日期：2026-07-02
- 项目：`/Users/chensiyang/dev/program/Personal-Notebook`
- 目标：为“知栖”设计一枚可用于 Tauri 桌面应用的应用图标，替换当前紫蓝闪电占位图标，并保持小尺寸可读。

## 1. 范围结论

本次采用“栖页 / 温润青绿”方案：

- 图标主体由白色页片、青绿色叶片、深色栖枝和一颗金色灵感点组成。
- 图标不包含中文、英文缩写或其他文字。
- 视觉气质偏安静、可信、长期陪伴，符合本地优先个人知识库的产品定位。
- 资产交付以 Tauri 桌面图标为主，同时同步浏览器 favicon。
- 保留现有图标文件名和 Tauri 配置引用，降低配置改动风险。

本次不采用以下方向：

- 不继续使用当前紫蓝闪电图标，避免偏效率工具或通用开发模板气质。
- 不使用复杂插画、真实鸟巢、书本堆叠等小尺寸难读的具象图形。
- 不在图标中放置“知栖”“Z”“Q”等文字或字母标识。
- 不把图标做成单一绿色或单一米色调，避免视觉单薄。

## 2. 设计目标

用户在 macOS Dock、Windows 任务栏、启动台、安装包和浏览器标签页中看到图标时，应能感受到这是一个安放个人知识、整理想法并持续沉淀的本地应用。图标需要比普通笔记应用更有记忆点，但不能显得喧闹或营销化。

优先目标：

1. 在 512px、128px、64px、32px 和 16px 下保持主体清晰。
2. 和“知栖”中文名建立隐喻关系，而不是只表达“笔记”。
3. 和当前应用的本地优先、个人知识库、页面树、块编辑等定位相符。
4. 适合长期使用，不依赖流行渐变或过强科技感。

## 3. 符号系统

### 3.1 白色页片

白色页片代表知识库中的页面、笔记和可编辑内容。形态偏轻，避免做成硬质文档图标。

### 3.2 青绿色叶片

青绿色叶片代表知识的生长、整理和自然聚合。叶片与页片交叠，使“页面”和“叶片”在视觉上合一，对应“栖”的安放感。

### 3.3 深色栖枝

底部深墨绿横枝代表“栖”的落点。它让图标从纯页面/叶片变成一个可以停靠思绪的位置，也增强 16px 和 32px 下的结构识别。

### 3.4 金色灵感点

右上方低饱和金色圆点代表被保存下来的灵感、星标或当前焦点。它只作为记忆点，不承担主要识别职责；在极小尺寸下可以弱化或省略。

## 4. 颜色规范

主色板：

- 背景暖米白：`#F9F4EA`
- 背景雾青绿：`#DCEFE6`
- 主体青绿：`#0E766E`
- 主体生长绿：`#378F5D`
- 栖枝深墨绿：`#233630`
- 灵感金：`#DDAE4E`
- 页片高光白：`#FFFFFF`
- 页片浅绿阴影：`#E9F4EC`

配色原则：

- 背景使用暖米白到雾青绿的轻渐变，保持温和但不陷入单一米色调。
- 主体以青绿为核心，呼应“知识生长”和本地安全感。
- 栖枝使用深墨绿提供小尺寸对比。
- 金色仅作低饱和点缀，避免抢走主体识别。

## 5. 构图规范

图标使用圆角方形底板，整体符合 macOS 和 Windows 桌面图标习惯。

构图约束：

- 画布基准为 512x512。
- 外层圆角方形留出安全边距，建议 x/y 从 32px 开始，宽高 448px，圆角约 106px。
- 主体符号位于视觉中心略偏上，避免在 Dock 中显得下坠。
- 右上金色点与主体保持足够间距，不能贴边或像通知角标。
- 16px 和 32px 导出版本需要适当加粗栖枝、简化页片内部阴影，保证识别。

## 7. 生成方式

优先使用可编辑 SVG 作为源图标，再从 SVG 导出各尺寸位图和平台容器格式。这样可以保证图标稳定、可审阅、可重复生成，也比直接使用单张 AI 生成位图更适合应用图标。

如需用 AI 图像生成辅助探索，只用于风格参考，不把不可编辑位图作为唯一源文件。

## 8. 验证要求

完成资产替换后至少验证：

1. 打开 `src-tauri/icons/icon.png`，确认 512px 图标和设计一致。
2. 检查 `32x32.png` 和 `64x64.png`，确认小尺寸主体可读。
3. 运行与 Tauri 配置相关的测试或构建检查，确认图标路径仍有效。
4. 若工具链允许，重新生成并检查 `icon.ico` 和 `icon.icns`。
5. 确认不提交 `src-tauri/target/`、`.app`、`.dmg`、`.msi` 或安装包产物。

## 9. 后续实现策略

实现时保持改动小而聚焦：

- 先替换 `icon-source.svg`。
- 使用现有 Tauri 图标生成流程或等效本地工具批量导出平台图标。
- 同步 `public/favicon.svg` 为同一视觉体系。
- 不改产品名、Tauri identifier、窗口标题或其他打包配置。
