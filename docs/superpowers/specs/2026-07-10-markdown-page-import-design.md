# Markdown 页面导入设计

## 目标

将单个 Markdown 文件导入为知栖中的一个新顶层页面，并尽量保留可编辑的正文结构与本地图片。

## 范围

- 通过现有导入菜单选择一个 `.md` 文件。
- 使用首个一级标题作为页面标题；没有一级标题时使用文件名。
- 将标题、段落、待办、无序列表、有序列表、代码块和 Markdown 表格转换为既有块类型。
- 将加粗、斜体、删除线、行内代码和网页链接转换为现有富文本；不支持的行内语法保留为文本。
- 相对路径本地图片会从 Markdown 所在目录读取、写入知栖资产库，并成为图片块。
- 图片不存在或无法读取时，保留原始 Markdown 行为正文文本，避免静默丢失内容。
- 导入完成后打开新页面。

## 明确不做

- 不导入整个文件夹或多个 Markdown 文件。
- 不下载远程图片。
- 不将指向其他本地 Markdown 文件的链接转换为知栖页面关系；该能力在单文件导入中无法可靠解析目标页面，已记录到待做清单。
- 不增加新的 Markdown 解析依赖；使用项目内最小解析逻辑，避免扩大桌面包体和维护面。

## 数据流

1. 选择 Markdown 文件，并获得文件名、文本与桌面端源路径。
2. 解析文本为页面标题、块描述和待导入的相对图片路径。
3. 逐张导入可读取的本地图片，失败时将对应图片描述降级为段落。
4. 使用现有 store 新建顶层页面并写入块，随后跳转至该页面。

## 验证

- 解析单元测试覆盖常见块、富文本、表格、标题回退和图片失败回退。
- 应用导入测试覆盖选文件、新建页面、跳转和相对图片写入。
- 运行相关前端回归与生产构建。
