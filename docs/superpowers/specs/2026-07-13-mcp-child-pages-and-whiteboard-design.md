# MCP 子页面同步与完整白板写入设计

## 目标

让本地 MCP 创建的子页面与桌面端创建结果一致，并让外部 AI 可以创建、读取和增量编辑知栖白板。

## 已确认的行为

- `create_page(parentId)` 必须原子地创建子页面，并在父页面末尾写入 `{ type: "child_page", pageId }` 块。
- 白板节点可以明确给出位置、尺寸、形状、颜色、线宽和层级；未提供时保留当前网格布局与默认样式。
- 白板连线可以独立指定起点/终点方向、锚点、直线或曲线、颜色、线宽以及两端标记。AI 根据图的语义自行选择，不再固定为右侧连到左侧。
- 白板写入支持笔画：颜色、线宽和坐标点。
- 白板编辑支持按元素 ID 擦除；读取白板后，AI 可以先确认元素 ID，再执行擦除或追加，避免按坐标误删。

## MCP 接口

### `create_page`

保持现有参数和返回结构。`parentId` 非空时，返回新增的 `parentBlockId`，用于确认父页面中的子页面块已同步创建。

### `append_content` 的 `whiteboard`

保留现有 `title`、`nodes`、`edges`，增加可选 `strokes`。节点和连线增加与桌面白板快照一致的可选样式字段。旧客户端只提交 `id`、`kind`、`text`、`x`、`y` 时，结果保持兼容。

### `get_whiteboard`

接收 `boardId`，返回白板标题和快照，供 AI 获取可操作的元素 ID。

### `update_whiteboard`

接收 `boardId`、可选的 `nodes`、`edges`、`strokes` 和 `eraseIds`。该调用在一个事务内追加新元素、擦除指定元素，并同步删除引用已擦除节点的连线。不会提供整板覆盖或清空操作。

## 数据与安全边界

- 继续使用现有白板快照结构，不新增前端白板格式或数据库表。
- 所有坐标、尺寸、颜色、线宽、连接锚点和笔画点都要校验；限制节点、边和笔画点数量，防止异常请求占用本地资源。
- 追加、擦除、审计、搜索索引与 MCP 工作区修订号必须在同一 SQLite 事务完成。事务提交后尽力发送桌面端刷新事件；通知失败不会回滚已提交的数据，客户端以 SQLite 中的最新快照为准。
- 擦除是用户明确授权的新 MCP 写入能力，仅允许按已读取到的元素 ID 删除；不提供按区域盲擦或全量清空。

## 验收

1. MCP 创建子页后，父页正文有可打开的子页面块，且任一写入失败均不留下半成品。
2. AI 可创建上→下和左→右两种图，连线方向、曲线/直线、颜色、标记与请求一致。
3. AI 可写入笔画，读取同一白板后可擦除指定笔画、节点或连线；擦除节点会清理关联连线。
4. 旧版白板 MCP 请求、现有桌面白板和完整测试套件保持通过。
