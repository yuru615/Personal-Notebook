# 页面回收站设计

## 目标

删除页面时先移入回收站，保留 30 天；用户可一次恢复该页面、全部子页面和原有关联资源。

## 方案

页面记录新增 `deletedAt` 与 `deletedRootId`。删除父页面时，整棵子树写入同一个删除根标识，但保留原有 `parentId`、块内容、白板、数据表、导图和媒体引用。正常页面树、路由与搜索只使用未删除页面。

回收站只显示每棵删除树的根页面。恢复操作清除该根及其子树的删除标记，因此能在原层级原样恢复。应用启动时清理删除时间已超过 30 天的整棵树；仅在此时执行现有资源引用过滤和孤儿资源清理。

## 界面范围

- 左侧顶部图标栏将“导出工作区”替换为“回收站”。工作区导出保留在已有的更多菜单。
- 新增 `/recycle-bin` 页面，显示删除树标题、删除时间、子页面数量与“恢复”按钮。
- 删除确认文案明确说明：页面会移入回收站，30 天内可恢复。

## 非范围

- 不做单项永久删除、批量清空或可配置保留期限。
- 不修改自动备份、崩溃恢复或另一工作树中的 Toggle/Callout 功能。
