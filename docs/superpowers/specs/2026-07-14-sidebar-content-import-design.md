# 左侧导入内容入口设计

**日期：** 2026-07-14
**状态：** 已获用户确认

## 目标

将紧凑侧边栏顶部的导入按钮改为“导入内容”。该入口仅用于把页面包或 Markdown 新增为页面；“导入完整备份”继续保留在更多菜单及其他现有位置，并维持覆盖警告。

## 决策

顶部按钮在桌面端打开一个同时支持 `.zhiqi`、旧 `.zip`、`.md` 与 `.markdown` 的文件选择框。按扩展名分流：Markdown 沿用现有 Markdown 页面导入；归档沿用现有页面包导入。若用户选到完整工作区备份，现有页面包校验会提示应使用“导入完整备份”，不会覆盖数据。

## 范围与验证

- 仅调整紧凑侧边栏顶部按钮及其回调。
- 保持页面菜单、设置中心和完整备份入口原样，避免扩大本次 UI 改动。
- 覆盖按钮文案、文件筛选条件、Markdown 与页面包分流的自动化测试。
