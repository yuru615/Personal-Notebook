# 自动备份与崩溃恢复设计

**日期：** 2026-07-15

## 目标

让桌面版知栖在本地异常退出后，能够让用户在下一次启动时选择恢复最近一次自动备份；同时在设置中心管理自动备份开关、频率和保留份数。

默认规则：自动备份启用；工作区发生变化后，每 15 分钟最多创建一次；仅保留最近 14 份。

## 范围

- 仅覆盖 Tauri 桌面运行时。
- 自动备份复用现有完整 `.zhiqi` 工作区归档，包含页面、白板、数据表、思维导图和受引用的附件。
- 自动备份文件保存到应用管理的本地备份目录，不写入工作区、页面树、导入内容列表或用户手动导出的文件夹。
- 设置中心的“通用”页增加“自动备份与恢复”卡片，包含：启用开关、频率选择、保留份数选择、最近成功时间和最近错误。
- 发生未正常结束的上一次运行时，在加载工作区前显示恢复选择：`恢复最新自动备份` 或 `正常打开`。

## 非目标

- 不新增云端同步、账号、网络传输或跨设备备份。
- 不增加历史备份浏览器、任意时间点恢复或手动删除单份自动备份；手动完整备份继续使用现有导出入口。
- 不为浏览器降级模式承诺自动备份或崩溃恢复。

## 方案

### 归档与保留

Rust 存储层直接调用现有完整工作区归档逻辑，并将字节写入应用数据目录的专用自动备份文件夹。文件先写入同目录临时文件，再以原子重命名发布；归档完整成功后才删除超过保留上限的最旧文件。因此写入失败、磁盘不足或进程中断都不会破坏已有备份。

自动备份不经 WebView 传输归档字节。调度器只在本次桌面运行中检测到工作区数据变更、距上次成功自动备份达到设置的间隔时运行。它同时覆盖本地编辑和 MCP 写入；仅改变应用设置不会单独生成工作区备份。

首版设置值固定为下列安全选项：

- 是否启用：开 / 关；默认开。
- 频率：15、30、60 分钟；默认 15 分钟。
- 保留份数：7、14、30 份；默认 14 份。

备份元数据保存在现有独立应用设置中：配置、最近成功时间和最近失败原因均不会随完整工作区导入或恢复覆盖。

### 异常退出检测

桌面进程启动时，在独立应用设置中写入“本次运行中”标记，并读取上一次的结束状态。应用收到正常退出事件后，在所有待保存操作完成后才写入“正常结束”。

如果下次启动仍发现上一次处于“运行中”，则视为可能异常退出。关闭窗口但隐藏到托盘不属于退出，不会触发恢复提示。强制结束、崩溃、断电或系统终止可能触发提示；这是宁可多提示一次也不静默丢失恢复机会的取舍。

### 恢复流程

1. 应用先检查是否存在有效的最近自动备份，再决定是否显示恢复弹窗。
2. 用户选择“正常打开”时，记录本次提示已处理，按现有工作区启动；不修改任何内容。
3. 用户选择“恢复最新自动备份”时，先尽力写出一份当前工作区的保护归档，再验证并导入最近自动备份。
4. 当前保护归档写入失败不阻止用户继续恢复，但会明确提示该风险；自动备份导入失败则保持当前工作区不变并显示错误。
5. 恢复成功后，重新加载工作区状态并进入正常页面；恢复提示不会在本次启动中重复出现。

恢复与现有手动“完整备份导入”使用同一套归档验证、资产完整性检查和事务替换逻辑，避免出现两套不一致的恢复规则。

### 设置中心与错误处理

设置中心采用现有应用设置保存路径。修改开关、频率或保留数量立即持久化；关闭自动备份会停止后续调度，但不删除已有自动备份。

自动备份失败不影响编辑、保存或 MCP 写入。设置中心显示最近失败原因；下一次满足间隔且发生数据变化时自动重试。清理仅发生在新归档发布成功之后，始终至少保留一份已成功写入的备份。

## 组件边界

- `src-tauri/src/storage/`：自动备份归档写入、列举、清理、恢复前保护归档和恢复调用；复用现有工作区归档函数。
- `src-tauri/src/lib.rs`：桌面生命周期标记与恢复状态命令注册。
- `src/lib/storageClient.ts`：自动备份与恢复状态的类型化命令边界。
- `src/lib/appSettingsRepository.ts`、领域类型与 store：自动备份设置及状态持久化。
- `src/app/App.tsx`：在工作区 bootstrap 前处理恢复选择，恢复后重新加载状态。
- `src/components/settings/`：通用设置页的自动备份与恢复卡片。

不新增网络服务、后台守护进程、第二个数据库或新的备份格式。

## 验证

- Rust：归档原子发布、14 份保留、失败不清理旧备份、完整归档含附件、恢复失败不替换当前数据、运行标记检测。
- 前端：默认设置、设置改动持久化、异常恢复弹窗的两种选择、恢复后工作区重载、错误提示。
- 回归：现有完整备份导入导出、MCP 写入、页面保存、关闭到托盘和正常退出。
- 发布前：完整前端测试、完整 Rust 测试、静态检查、生产构建和 Windows NSIS 安装包验证。
