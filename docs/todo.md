# 待做功能

本文件记录已经确认“值得做”，但当前轮次不适合一起塞进去的功能。

原则：

- 只记已经讨论过、并且明确暂缓的项
- 每条都写清楚为什么先不做
- 当前轮次做完后，如果仍然成立，就保留；如果已经顺手完成，就移除

## 编辑器多选块

### 1. 多选后的浮动小工具条

- 状态：暂缓
- 原因：多选块第一版先只做最小闭环，避免和批量删除、整组拖动一起把交互面铺太大
- 预期方向：在框选完成后，提供靠近选区的轻量工具条，承载批量删除、批量转换、后续更多批量操作

### 2. `Shift` 连选、`Ctrl / Cmd` 增减选

- 状态：暂缓
- 原因：当前先验证鼠标框选主路径，避免第一版就把多种选区规则揉在一起
- 预期方向：补齐桌面编辑器常见的键盘选区能力，并和鼠标框选共存

## 本地工作流

### 1. 全局快捷键速记

- 状态：暂缓
- 原因：需要处理系统级快捷键注册、独立小窗口、窗口焦点和快捷键冲突；不应与文件拖拽导入同时改动桌面窗口生命周期
- 预期方向：在任意软件中通过可配置快捷键打开极小速记窗口，提交后默认进入收件箱

### 2. 自动备份与崩溃恢复

- 状态：暂缓
- 原因：涉及备份频率、保留策略、磁盘空间和恢复入口，属于数据安全能力，需单独设计与验证
- 预期方向：定期生成可恢复的本地工作区备份，并在异常退出后提供一次明确的恢复选择

### 3. PDF 预览与原文件联动

- 状态：暂缓
- 原因：PDF 预览和“用系统默认程序打开原文件”分别涉及预览能力和桌面文件安全边界，不应影响当前 Markdown 页面导入闭环
- 预期方向：PDF 提供预览与后续全文检索；附件可显示原文件路径并调用系统默认程序打开

### 4. Markdown 跨文件链接转页面关系

- 状态：暂缓
- 原因：单文件 Markdown 导入无法可靠确定其他本地 `.md` 文件是否已导入、对应哪个页面，以及文件移动后的引用如何恢复
- 预期方向：批量导入同一目录或显式选择多份 Markdown 后，建立源文件路径到页面 ID 的映射，再把相对 `.md` 链接转换为知栖页面链接

### 5. Markdown 远程图片下载入库

- 状态：暂缓
- 原因：需要明确联网授权、超时与失败提示、文件大小限制和远程地址失效后的处理，不应让单文件导入在无提示下产生网络请求
- 预期方向：在 Markdown 导入设置中允许用户选择“下载远程图片到本地资产库”，成功后转换为知栖图片块；失败时保留原始网页图片链接

## 外部 AI 接入

### 1. 网络 API 与云端同步接入

- 状态：暂缓
- 原因：网页、移动端或远程 AI 写入本地知识库，需要同时解决账户身份、设备授权、密钥管理、设备在线、内容同步冲突与隐私边界；当前本地优先架构不应为此过早引入云端服务
- 预期方向：在本机 MCP 接入稳定、且明确需要跨设备或网页/移动端 AI 保存内容后，再设计受 OAuth/设备授权保护的网络 API 与同步层；API 复用统一内容导入契约，支持文本、资产、白板、思维导图和数据表
